| 2 | 플레이어 행동 설계   Player Action Design |
| :---: | :---- |

**플레이어는 무엇을 하는가? (핵심 행동 동사)**

*명사가 아닌 동사로 서술 — 예) '자원을 배분한다', '서류를 검토하고 승인·거부한다'*

플레이어의 주요 행동을 동사 중심으로 기술하세요.

의심한다: 상대방이 제시한 물건의 가격과 현재까지 쌓인 '좋아요' 토큰의 개수를 대조하여 신뢰도를 판단한다.  
허위 매물을 제시한다: 빌런으로서 벽돌(꽝) 카드를 비싼 노트북인 것처럼 속여 높은 가격에 판매를 제안한다.  
신고/평가한다: 거래 결과가 만족스럽지 않을 때 상대의 평판 토큰을 소멸시켜 시장에서 퇴출시킨다.

| 어떤 선택을 하는가? 반복적으로 내리는 의사결정 물품 등록 (판매 매물 등록)  시장의 수요와 공급에 맞춰 가격 설정  높은 수요에도 적정 가격을 유지할 경우 \-\> 긍정적 평가 획득 수요를 핑계로 과도하게 가격을 인상할 경우 \-\> 부정적 평가 획득 거래 제안 및 협상 플레이어의 보유 자산 한도 내에서 협상 진행 거래 실행 및 의심 판정 상호 합의된 가격으로 거래 진행 시세 대비 가격이 비정상적으로 높을 경우 \-\> 사기 의심  시세 대비 가격이 비정상적으로 낮을 경우 \-\> 사기 의심 거래 결과 확인 및 평가/신고 원활하게 협상 및 거래가 완료된 경우 \-\> 긍정적 평가 부여 사기 물품(벽돌 등) 배송이 발각된 경우 \-\> 즉시 평판 하락 조치 빌런의 블러핑 개입 \-\> 정상 물품을 받았음에도 사기당했다고 거짓 신고 가능 평판 0 도달 시 시장 퇴출 플레이어 중 한 명이라도 평판이 0이 되면 게임 종료 빌런의 평판이 먼저 0이 될 경우 시민 승리 시민의 평판이 먼저 0이 될 경우 빌런 승리 |
| :---- |

**성공 조건 / 실패 조건**

| ✓ 성공 조건 플레이어가 목표를 달성하는 조건 【시민】 빌런의 평판을 0으로 만들어 색출하고, 게임 종료 시점에 가장 많은 자산을 보유한다. 【빌런】 ①모든 라운드가 끝날 때까지 정체가 밝혀지지 않는다. ②시민보다 많은 자산을 보유하면 승리한다.  | ✗ 실패 조건 게임이 실패하거나 종료되는 조건 빌런이 아닌 시민의 평판이 먼저 0이 되어 게임이 종료되거나, 빌런을 잡지 못하고 자산을 잃는다.  |

| 3 | 핵심 메카닉 설계   Core Mechanic Design |
| :---: | :---- |

**핵심 메카닉 (1개, 한 문장)**

*게임의 가장 중요한 규칙 하나를 정의하세요. 이 메카닉이 문제 해결과 직접 연결되어야 합니다.*

예) 플레이어는 제한 시간 안에 서류 정보를 비교하여 입국자를 승인하거나 거부한다.

플레이어는 타인과 거래 하며 비공개 카드 교차 검증 및 상호 평판 피드백을 통해 빌런을 식별하고 시장에서 자산을 획득한다.

| 행동 → 규칙 변환 *플레이어 행동을 게임 규칙으로 어떻게 공식화하는가* 예) 알림 발생 → 무시/확인 선택 → 집중 게이지 변화 판매 행동 발생 → 구매자 선정 및 카드 제출(쿨거래/취소) → 결과 확인(자산 변동) → 상호 평판 평가(좋아요 증감).    | 반복 가능한 루프 구조 *한 사이클의 흐름을 단계적으로 기술* 예) 자극 발생 → 선택 → 결과 반영 → 다음 자극 → 반복  행동 카드 뽑기 → 거래 제안 → 대화 및 협상 → 거래 실행 → 평판 피드백 → 다음 턴 진행.    |
| :---- | :---- |

| 문제 해결과의 연결 고리 *이 메카닉이 어떻게 정의된 문제를 해결하는가* 메카닉이 실제 문제 해결 경험을 어떻게 만들어내는지  메카닉이 '이기적 행동이 장기적인 거래 기회를 박탈한다'는 인과관계를 보여줌으로써 실제 중고거래의 신뢰 문제를 경험하게 한다. | 보조 메카닉 (선택) *핵심 메카닉을 보완하는 추가 규칙* 예) 자원 관리, 시간 압박, 정보 비대칭, 협력 구조 등 ① 정보 비대칭: 각 플레이어는 자신의 카드 내용만 알며, 상대 카드는 거래 완료 후에야 공개된다. ② 평판 토큰 관리: 쿨거래 시 \+1, 노쇼·벽돌 거래 시 \-1이 적용되며, 0이 되면 즉시 시장에서 퇴출된다. |

**경험의 흐름 (시간 순서)**

*게임 시작부터 끝까지 플레이어의 감정·경험이 어떻게 변화하는가*

예) 호기심 → 집중 → 압박감 증가 → 선택의 딜레마 → 결말의 충격 또는 성취감

호기심(초반 거래) → 의심(벽돌 등장) → 압박 증가(평판 위기) → 딜레마(최후의 선택) → 성취 또는 충격(결말).

| 핵심 기준 자가 체크 (Core Criteria) 이 게임은 어떤 문제를 어떻게 해결하는가? 중고거래의 정보 비대칭 문제를 '평판 토큰'과 '비공개 검증 메카닉'을 통해 시각화하고, 플레이어가 스스로 신뢰를 구축하며 문제를 해결하게 한다. 해결 과정에서 어떤 딜레마를 경험하는가? *플레이어가 겪는 핵심 갈등 구조* A를 선택하면 B가 희생되는 구조 설명  미션 달성(단기 이득)을 위해 비매너를 저지를 것인가, 아니면 원활한 거래(장기 생존)를 위해 매너를 지킬 것인가 사이에서 갈등한다.  |
